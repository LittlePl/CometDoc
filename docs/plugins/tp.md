# 传送系统 <Badge type="tip" text="生存"/> <Badge type="tip" text="模组"/> <Badge type="tip" text="空岛"/>
## 简介
传送系统为玩家提供一系列传送功能，包括玩家间传送、返回死亡点等，可满足玩家基本的传送需求。**Comet 生存服务器**和 **Comet 模组服务器**具有这个功能。部分服务器可能会根据实际情况对玩家提供的功能进行调整或删减。

## 功能一览
- **玩家间传送**（/tpa）：玩家可付费传送至另一名在线玩家，或将该在线玩家传送至自己。
- **地标传送**（/warp）：可选择一个管理员已事先配置好的地点传送。
- **家园系统**（/home）：又称**定点传送**，玩家可自行创建一个传送点以便将来进行传送。
- **返回死亡点**（/back）：玩家可付费传送至最近一次的死亡处。

## 注意事项
1. 当使用玩家间传送时，始终是**发送请求方**扣除经济。

2. 当使用玩家间传送时，若接收方处于别的UI界面（如打开聊天栏、打开物品栏）中，且能收到请求UI，该请求只能通过指令完成。

3. 玩家可以自行配置是否接收传送请求的UI界面。如果关闭此功能，当其他玩家向您发送传送请求时，将不再弹出接受或拒绝页面。您只能使用`/tpaaccept`同意请求，或使用`/tpadeny`拒绝请求。
