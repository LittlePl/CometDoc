# 个性化图标
## 什么是图标
图标是服务器菜单页面中的一种表现形式，主要显示在每个按钮旁边的位置。

在实际应用中，合适的图标能够显著提升玩家浏览服务器菜单的体验，吸引眼球，带来强烈的视觉冲击力。

## 实际应用
- **服务器菜单**：图标可以美化菜单，提高观赏性。
- **Comet 生存服务器的玩家商店**：玩家上架的物品会自动显示对应的图标，对于买家来说，浏览商品一目了然，可以提高玩家购买物品的效率。除此之外，玩家店铺也具备图标，并且玩家可自定义。
- **Comet 生存服务器的联盟系统**：在所有联盟页面中显示，可适当提高联盟显眼度和威望，吸引更多小伙伴前来观望你的联盟。

## 使用指导
服务器的部分功能，如**联盟系统**和**商店系统**，有些地方可以由玩家来自定义。我们提供了一份使用指导，供玩家们参考。

- 图标支持使用**本地路径**和**url链接**。
- 图标本质上是显示一个图片（或贴图），因此可指向任何一个图片。
- 若使用本地路径，需要以`textures/`作为开头，然后参考资源包路径进行编写。
- 若使用本地图标，**该路径指向的图片需要存在于所有玩家的游戏中，才能对每个玩家都正常显示。**例如：设置联盟图标默认使用的资源包标准路径为`textures/ui/mashup_world`。
- 如果本地图标的路径指向了一个不存在的图片，则会显示成紫黑格。
- 玩家可通过前往 github 下载[“bedrock_samples”](https://github.com/Mojang/bedrock-samples "`bedrock_samples`")资源包了解游戏中各个图标的位置，并应用在任何能自定义图标的地方。
- 若使用**url链接**，该链接需指向图片直链，否则图标位置将在加载很长时间后自动消失。
- 图标使用示例：
	- `textures/items/apple_golden`： 金苹果
	- `textures/items/slimeball`：粘液球
	- `textures/blocks/amethyst_cluster`：紫水晶方块
	- 图标只能使用图片格式。部分图标以特殊格式储存（如.tga），因此不可用。
